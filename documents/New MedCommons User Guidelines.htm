Congratulations. If you are reading this then you are in some way part of the extended MedCommons family and we have collectively decided that you should be collaborating with us online. Here’s what you have to do:

 

Get a Yahoo Messenger ID and client

 

We recommend you get yourself a Yahoo Id for instant messaging as we often have online multi-party discussions. Go to www.yahoo.com and sign up for a Messenger account. Download the client and get comfortable. You will undoubtedly end up with a separate Yahoo mail account. You can save that for your magazine and newsletter subscriptions, as MedCommons will not use your Yahoo account for anything except our group conferences. Once you are on you should contact bdonner@medcommons.org with your ID, etc, and we will get you going with the other group Ids.

 

Here’s a list of the ID’s for the others in the group. Add your name to the list after you learn how to use Subversion (see below). You are not done with setup until everyone can see you on this list.

 

Get a MedCommons Email Address

 

We will set up an email account for you as jdoe@medcommons.org assuming you are John or Jane Doe. We’ll eventually be moving to medcommons.net, but for now use this. Please let us know whether you want you want a real mailbox or you want your mail forwarded to an existing mailbox. Unless we hear otherwise we’ll assume you want it forwarded to the email address we already have for you.

 

If you want a real mailbox you will be given a Pop3 mail account and you will have to tell your email software about it:

 

Incoming Pop3:  pop.1and1.com

Outgoing Smtp: smtp.1and1.com

Account: m12345678-6 (we will give you this in an email to your old account or over IM)

Password: xxxxxxxxx   (likewise)

 

If we have created a mailbox for you then you can also use it via the Web. It’s a bit crude, but if you go to webmail.perfora.net and put in your account id and password you can use the mail system from the road.

 

Get Connected to the Mantis Bug and Issue Tracking System

 

Now you need to get connected to Mantis, which is our issue and bug tracking system.

This is quite easy. Just point your browser to www.medcommons.net/mantis/

 

Your name and password will be the front part of your email name (e.g. jdoe). Change your password as soon as you get on.

 

Get Connected to the MedCommons Calendar

 

Likewise for our calendar system.  Go to our calendar pages at www.medcommons.net/webcalendar and have at it. You will see that we have some corporate calendars, and you can create your own calendar as well. Do not complain that you can not synch up with iCal or Outlook, etc, but if you try really hard and succeed please let the rest of us know.

Get Connected to the Subversion Repositories you need to access

 

Next, you need to connect to our document repositories, which is maintained via Subversion. You will receive an email at your jdoe@medcommons.org address with your password for each separate repository. It will be very cryptic. The repositories have URLs that look like:

 

www.medcommons.net:6666/svn/documents - everyone

 

www.medcommons.net:6666/svn/hr - accountants, lawyers, payroll

 

And so forth. Set these up and make sure that you can move around in these trees from your web browser. The top level of each repository contains an Index file that has a description of what’s in that tree and how to get around.

 

It is absolutely essential that this work for as this is our primary means of collaboration. You can’t update any documents this way, but much of the time you just want to look at something over the Web.

 

All of these web resources take a bit of remembering. We strongly recommend that you not only record them on your favorites list, but that you also store them in some web-based site. For example, if you set up a Yahoo Messenger account you can make a personal page and record all of these sites as bookmarks and get back to all of them via my.yahoo.com

Get Tortoise Shell Extensions for Windows Explorer

 

Now comes the hard part: If you are a real contributor you will need read and write access to the subversion repositories. We are only supporting access via Windows 2000 or XP. You need to download some Windows Explorer extensions called Tortoise. Here’s how to do it:

 

Go to:

http://tortoisesvn.tigris.org/download.html

take Version 1.0.3.968 for NT4/Win2k/XP

Download this file (e.g. right click, Save As) to your desktop.

Doubleclick on the file and it will install Tortoise.

 

Once Tortoise is up and running you will need to connect to the various repositories:

 

Invoke Windows Explore (right click on Start button in lower left of your screen, Explore All Users)

 

Congratulations again. Tortoise is installed. Now it needs to be configured once for each repository you want to connect to:

 

 

  • Make a folder for yourself. This is where your personal copies of the files in the subversion tree will be stored. It doesn’t matter much where, but make sure you can get to it easily, as you will be spending a lot of time there. Make a shortcut to this folder and put it on your desktop and get it into your Start menu if you know how.

 

  • Let’s assume you put the folder at C:/SubversionDocs/

 

  • Now go to Windows Explorer and click on File > Checkout

The URL of the repository is the URL you were sent in the email (e.g. www.medcommons.net:6666/svn/documents)

The Checkout directory is C:/SubversionDocs/

 

  • Make sure the Head Revision radio button is selected. Hit enter, you will be prompted for user name and password which you got in the email. Tortoise will start doing lots of good things. Go get a cup of coffee.

 

Repeat this procedure for each repository, creating a separate Checkout directory folder for each. Make sure each folder is free standing and not in the middle of another subversion tree. Just putting them all at C:/ or in My Documents/ will do that.

 

Once connected to the repositories you can access all the files as normal files. Periodically you will want to make sure you have the latest version of a file. Use the somewhat awkwardly named Update command to get the latest changes from others. If you make changes yourself, use the Commit command to get them back to subversion.

 

To create a totally new file, use your favorite application, put it in the right directory, use the Add command to tell subversion about the new file, then use  Commit to push the files back to subversion. That’s all. There’s a really chunky Subversion manual  and other docs and FAQs to read if you are so inclined.
